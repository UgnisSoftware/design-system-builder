Ugnis is a new type of framework

Current JavaScript ecosystem is frustrating. Tools like JSX and Observables try to reduce boilerplate, but they don't solve the underlying problem - all current solutions are a one-dimensional string of symbols:

  • Text cannot show how your different parts of your code work together
  • You have to know the how a valid program looks before you can start writing
  • Text is prone to errors and easily tangles together

Ugnis is a visual front-end framework

  • There is no new language to learn
  • Setting up a new project takes seconds not hours
  • Generated code is guaranteed to have no runtime errors
  • Understand how code works by poking around
  • Forget obscure HTML, CSS and JavaScript rules

What Ugnis is aiming to be:

For Front-end developers

  • Full framework with view, state management, styling, routing and http requests
  • Declarative, functional and type safe without getting in your way
  • Familiar - single state container, one way data flow, functional rendering
  • Starting a new project is instant, no build tools or configuration
  • Can be introduced in small steps to an existing app
  • Faster and much smaller than any other framework
  • Support multiple people working on the same project is not yet implemented
  • Saving your application is not possible in beta
  • Text-based tools, like Git, are not build to support visual programming languages

For people without JavaScript or any programming experience

  • As powerful as a programming language - create any webapp you can think of
  • A way to avoid the notorious JavaScript ecosystem
  • Simple - become productive in a few hours
  • Guaranteed safety - can be learned and used without the fear of making mistakes
  • Full development environment that works on mobile
  • Encourages everyone on the team to contribute
  • Not a back-end (data storage) provider for your apps

How do I use Ugnis?

You use the visual editor to create the definition of your application. Once your app is ready, you can export a minified JavaScript code and put it in your website.

How do I save my application?

Ugnis is changing rapidly and we cannot support saving applications before we release 1.0v

How does Ugnis work internally?

Ugnis is declarative programming taken to the extreme - code is just a JSON file. Other parts are:

  1. Interpreter for the JSON.
  2. Editor, because you wouldn't want to write JSON by hand. Editor works with the Interpreter to change your definition JSON.
  3. Compiler that takes your JSON and makes it into an optimized JS code that does not require an interpreter.

Can Ugnis create an app for Windows, Linux and Mac?

Yes, Ugnis 1.0v will allow automatic wrapping with Electron.

Can Ugnis create and iOS or Android app?

Not yet, though in the future a slightly changed editor with more specialized components will be able to export to iOS and Android.

Visual programming will never work.

That is not a question and it already works. Ugnis did not start as a visual tool. It became obvious that we needed more than one dimension to express the cyclical nature of applications.
